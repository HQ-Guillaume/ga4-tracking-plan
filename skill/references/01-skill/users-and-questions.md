# Users And Questions

Use this file to keep the skill focused on the people and questions it serves.

## Users

Primary users:

- web analysts;
- analytics consultants;
- tracking-plan owners;
- analytics engineers;
- agencies and internal analytics teams using AI agents to prepare or review
  tracking plans.

Secondary users:

- developers and dataLayer owners implementing the plan;
- QA and recette teams testing GTM Preview, DebugView, and network requests;
- media and performance teams depending on conversion and audience signals;
- product, ecommerce, CRM, or content managers reading the measurement logic.

## Questions The Skill Resolves

- What should be tracked for this website, page, or journey?
- Which GA4 automatic, enhanced-measurement, recommended, or ecommerce events
  should be used?
- When is a custom event justified, and what official alternatives were
  considered?
- Which parameters are needed, what values are allowed, and which custom
  dimensions or metrics should be registered?
- Which events belong to the same journey and how can users read that journey
  easily in the Event Matrix?
- What should not be tracked because it is noisy, duplicate, unavailable,
  sensitive, or not actionable?
- What assumptions, blockers, privacy risks, and QA checks exist?
- Can the plan scale to future pages, markets, journey variants, or automated
  recette?

## Successful Answer

A successful answer is a coherent journey-level measurement model, not a list
of disconnected events. Events and parameters should work together around the
business goal and analysis needs, while staying readable for implementation and
QA.
